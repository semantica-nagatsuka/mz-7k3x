# シフト管理システム 機能仕様書

> バージョン: 1.2  
> 対象: 自店舗試験運用フェーズ  
> 最終更新: 2026年3月

---

## 1. システム概要

### 背景

水商売店舗のキャストシフト管理をLINE個別連絡＋手入力Excelから脱却するための、軽量Webシステム。  
キャスト同士のLINE交換禁止の制約により、LINEグループ・botは利用不可。

### 技術構成

| 項目 | 内容 |
|---|---|
| フロントエンド | HTML + JavaScript（単一ファイル） |
| データベース | Firebase Realtime Database |
| ホスティング | GitHub Pages（無料） |
| フレームワーク | 不使用（ライブラリなし） |
| キャスト用 | `index.html` |
| スタッフ用 | `admin.html` |

### 非機能要件

| 項目 | 仕様 |
|---|---|
| 対応デバイス | スマートフォン最適化（PC閲覧も可） |
| デザイン | 店舗ごとにアクセントカラー・ロゴを設定ファイルで変更可能 |
| 保守性 | 単一ファイル構成・フレームワーク不使用でバグ発生リスクを最小化 |
| 拡張性 | 基本的にカスタマイズ不可。店舗カラー・ロゴのみ対応 |
| カスタマイズ対応 | CSV並び順変更等は都度有料対応（目安：3,000円〜） |
| ドメイン | 現状はGitHub Pages固定URL。外部展開時に独自ドメイン取得予定 |
| セキュリティ | キャスト名＋PINハッシュをFirebaseセキュリティルールで直接照合。ブラウザの認証情報に依存しないためITPの影響を受けない |

### ビジネス展開予定

| フェーズ | 内容 | 料金 |
|---|---|---|
| α | 自店舗試験運用（約2ヶ月） | 無料 |
| β | 同一グループ店舗展開（最大30店舗） | 月3,000円／店舗 |
| 正式 | 外部展開（ナイトワーク業界他社） | 月10,000円／店舗 |

### 価格根拠

|---|---|
| シフト管理SaaS（市販品） | 月1,000〜3,000円／人 ＝ 15人規模で月1.5〜4.5万円 |
| LINE bot系（外注開発） | 初期費用5〜15万円＋LINE公式アカウント月5,000〜15,000円 |
| 本ツール | 月10,000円／店舗（グループ内3,000円） |

## 2. 運用フロー

| ステップ | 担当 | 内容 |
|---|---|---|
| URL配布 | スタッフ | 個別LINEでURLをキャストに送付 |
| シフト入力 | キャスト | スマホでURLにアクセスして入力・提出 |
| データ確認 | スタッフ | admin.htmlで全員分の提出状況を確認 |
| CSV出力 | スタッフ | admin.htmlからCSVをダウンロードして管理 |
| 締切管理 | 自動 | 水曜25:00締切（管理画面から変更可能） |

## 3. 週の定義

### 基本構造

| 項目 | 仕様 |
|---|---|
| シフト週 | 月曜〜日曜 |
| 公式締切 | シフト週の**前々週水曜 25:00（木曜1:00）** |
| 遅延受付期間 | 締切翌日（木曜）〜 シフト週前週**土曜 23:59** |
| 自動ロック | シフト週前週**日曜 0:00** に自動ロック（入力不可） |

### 具体例

> 対象週：3/24（月）〜 3/30（日）の場合

```
公式締切:   3/12（水）25:00
遅延受付:   3/13（木）〜 3/22（土）23:59
自動ロック: 3/23（日）0:00
```

### ステータス定義

| ステータス | 期間 | キャスト画面 |
|---|---|---|
| 受付中 | 〜公式締切（水曜25:00） | 通常表示 |
| 遅延受付 | 木曜〜土曜 23:59 | 「締切済み」バナー表示。提出は可能 |
| 受付終了 | 日曜 0:00 以降 | 入力不可（ロック表示） |

### 週選択（プルダウン）

| 項目 | 仕様 |
|---|---|
| 選択可能範囲（過去） | 直前週まで |
| 選択可能範囲（未来） | 8週先まで |
| デフォルト表示 | 直近の未提出週（または今週） |
| 用途 | まとめて複数週分の提出が可能 |

## 4. データ管理

### 再提出・ロック

| 状態 | 挙動 |
|---|---|
| ロック前 | 上書き可（最新の提出が有効） |
| ロック後 | 提出ボタン非活性。変更不可 |
| ロック操作 | adminがキャスト別に手動ロック／解除 |

> **要検討：** 週単位の一括ロック機能の要否（「この週の全員をロック」ボタン）

### データ保持期間

| 項目 | 仕様 |
|---|---|
| 保持期間 | 該当シフト週の終了後（日曜）まで |
| 削除方式 | admin画面から手動削除 |
| 自動削除 | 非対応（Firebase Functions が必要なため後回し） |

## 5. キャスト画面（index.html）

### 画面構成

```
┌─────────────────────────┐
│ お知らせ（複数件・先頭表示）  │
├─────────────────────────┤
│ キャスト名 / 対象週選択      │
├─────────────────────────┤
│ 日別入力（月〜日）           │
│ 一括入力ボタン               │
├─────────────────────────┤
│ 週全体備考                  │
│ 提出ボタン                  │
└─────────────────────────┘
```

### お知らせ表示

| 項目 | 仕様 |
|---|---|
| 表示位置 | 画面先頭 |
| 件数 | 複数件同時表示 |
| 重要度 | 緊急 / 通常 の2段階（表示色・順序で区別） |
| 非表示 | キャストが確認ボタンを押したらその端末では非表示 |
| 既読管理 | なし（adminは投稿・削除のみ） |
| 将来拡張 | 運用開始後に要件が出たら拡張を検討 |

### 入力項目

#### 基本情報

| 項目 | 仕様 |
|---|---|
| キャスト名 | テキスト入力。初回入力後はlocalStorageに保存し次回自動入力 |
| 対象週選択 | プルダウン（過去1週〜未来8週） |

#### 日別入力（月〜日、7日分）

| 項目 | 仕様 |
|---|---|
| 出勤可否 | 選択（出勤可 / 休み / 保留） |
| 開始時間 | 12:00〜23:00、30分刻み（出勤可のみ入力可） |
| 終了時間 | 13:00〜25:00、30分刻み（出勤可のみ入力可） |
| 日別備考 | テキスト入力 |

未入力（未選択）のままの日は、データ上 `status: null` として送信される。

#### 一括入力ボタン

| ボタン | 動作 |
|---|---|
| 全日休み | 理由入力ダイアログを表示 → 入力後、7日分を「休み」に一括設定 |
| 全日保留 | 理由入力ダイアログを表示 → 入力後、7日分を「保留」に一括設定 |

**一括入力フロー**

```
ボタン押下
  → 理由テキスト入力ダイアログを表示（入力必須・空欄では適用不可）
  → 個別入力済みの日が存在する場合は上書き確認を追加表示
  → 適用：7日分の出勤可否を一括設定 ＋ 理由を週全体備考に自動挿入
```

- 「全日出勤可」ボタンは設けない（個別入力を基本とする）
- 一括ボタンと提出前補完は画面上で明確に区別する（UI詳細は実装時に検討）

#### 提出前の未入力日補完

提出ボタン押下時、未入力（`status: null`）の日が存在する場合に発火する。

| 項目 | 仕様 |
|---|---|
| 発火タイミング | 提出ボタン押下時 |
| 動作 | 未入力日の扱いをキャストに選択させる |
| UI形式 | 要検討（一括ボタンと紛らわしくない形にする） |
| 選択肢 | 休み / 保留 / そのまま提出（null のまま） |

#### 週全体

| 項目 | 仕様 |
|---|---|
| 週全体備考 | テキスト入力（一括ボタンの理由が自動挿入される場合あり） |

### 提出・送信

| 項目 | 仕様 |
|---|---|
| 送信先 | Firebase Realtime Database |
| タイムスタンプ | 送信時刻を自動付与 |
| 提出前確認 | 入力内容を確認してから送信 |
| 提出後 | 入力内容をその場でダウンロード可能 |
| 再提出 | ロック前は上書き可能 |

### 端末ID管理・データ復元

| 項目 | 仕様 |
|---|---|
| 初回入力時 | キャスト名＋4桁PINを設定して提出。PINはハッシュ化してFirebaseに保存 |
| データキー | `storeId / castName` をキーとして管理 |
| 復元フロー | キャスト名を入力 → Firebase上にデータが存在すればPIN入力を要求 → 一致した場合のみ過去データを表示・編集可能 |
| 編集・再提出 | ロック前であればそのまま編集・再提出可能 |
| ロック後 | 入力欄グレーアウト、提出ボタン非活性（読み取り専用） |
| 過去履歴表示 | 週選択プルダウンと連動し、提出済み週のデータを表示 |
| 別端末・localStorage消去時 | キャスト名＋PINを入力すれば自己復旧可能。サポート不要 |
| PIN忘れ時 | adminがデータを手動削除 → キャストが新規入力し直し（過去データは消えるが直近週のみなので実害は限定的） |
| 覗き見・改ざん防止 | PIN不一致の場合はデータを表示しない。他キャストのシフト閲覧・上書きを防止 |

### アラート・UI

| 項目 | 仕様 |
|---|---|
| 締切アラート | 締切日（水曜）が近づいたら表示 |
| 遅延バナー | 締切超過後は「締切済み・遅延提出」バナーを常時表示 |
| ロック後 | 入力欄グレーアウト、提出ボタン非活性 |

## 6. admin画面（admin.html）

### 必須機能一覧

| 機能 | 詳細 |
|---|---|
| 認証 | パスワード入力画面。正解でlocalStorageにフラグ保存。パスワードはFirebaseのsettingsで管理・変更可能 |
| お知らせ管理 | 投稿・削除。重要度（緊急／通常）を設定可能 |
| 週別フィルター | プルダウンで表示週を切り替え |
| 提出状況一覧 | 選択週の提出済みキャスト一覧を表示 |
| 締切超過ハイライト | 公式締切（水曜25:00）を超えた遅延提出を赤表示 |
| キャスト別ロックボタン | 個別にロック／解除。ロック後はキャストの編集不可 |
| CSV出力 | 選択週のデータをCSV形式でダウンロード |
| 設定 | 締切曜日・時刻、週開始曜日をFirebaseに保存（全端末に反映） |
| データ削除 | 週単位削除 ＋ キャスト個別削除。ロックボタンの横に個別削除ボタンを配置 |

### 未提出者の管理

名簿機能はシステム内に持たない。未提出者の確認は以下の運用で行う。

1. CSVをダウンロード
2. 手元のExcelマスターと照合（目視 or VLOOKUP）

### CSVフォーマット（暫定）

```
キャスト名, 対象週, 曜日, 日付, 出勤可否, 開始時間, 終了時間, 日別備考, 週全体備考, 提出日時, ロック状態
```

> **要確認：** 既存Excelの列構成を確認後、フォーマットを調整する

## 7. Firebase データ構造（暫定）

```json
{
  "stores": {
    "{storeId}": {
      "name": "店舗名",
      "adminPassword": "xxxxxxxx",
      "settings": {
        "weekStartDay": 1,
        "deadlineDay": 3,
        "deadlineHour": 25
      }
    }
  },
  "notices": {
    "{storeId}": {
      "{noticeId}": {
        "body": "今週X日に人が足りません！",
        "priority": "urgent",
        "createdAt": "2026-03-20T10:00:00+09:00"
      }
    }
  },
  "shifts": {
    "{storeId}": {
      "{weekStartDate}": {
        "{castName}": {
          "castName": "キャスト名",
          "pinHash": "ハッシュ化されたPIN",
          "weekStart": "2026-03-24",
          "submittedAt": "2026-03-20T15:30:00+09:00",
          "isLocked": false,
          "isLate": false,
          "weekNote": "週全体備考（一括入力時の理由も入る）",
          "days": {
            "2026-03-24": {
              "status": "available",
              "startTime": "18:00",
              "endTime": "24:00",
              "note": "日別備考"
            },
            "2026-03-25": {
              "status": null
            }
          }
        }
      }
    }
  }
}
```

**notices.priority の値**

| 値 | 意味 | 表示 |
|---|---|---|
| `urgent` | 緊急 | 赤・上部に優先表示 |
| `normal` | 通常 | 標準表示 |

**shifts.days.status の値**

| 値 | 意味 |
|---|---|
| `available` | 出勤可 |
| `off` | 休み |
| `pending` | 保留 |
| `null` | 未入力（提出前補完をスキップした場合） |

## 9. 対応外・制約事項

- キャスト同士のLINE交換禁止のため、LINEグループ・bot は非対応
- 名簿機能なし（未提出者判定は手元Excelとの照合で運用）
- 別端末からのアクセス時、提出履歴の引き継ぎ不可（新規扱い）
- 機種変・localStorage消去時はキャスト名＋PINで自己復旧可能
- 退店者が出た週は、adminが当該週のデータを手動削除する（同名新人との混同防止）
- PIN忘れ時の運用フロー：
  - 未ロック週の場合：adminがキャスト個別削除 → キャストが新規入力し直し
  - ロック済み週の場合：削除・再入力はさせない。adminがシフト内容をLINEで直接回答する
- 既存店舗管理システムとのCSV連携はシステム側の対応状況による
